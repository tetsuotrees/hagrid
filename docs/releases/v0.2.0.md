# Hagrid v0.2.0

Release date: 2026-03-07

## Summary

Hagrid v0.2.0 completes the v0.2 milestone with four new command families and
an opt-in notification system. The release adds policy-based auditing, file
change monitoring, interactive secret rotation, and webhook notifications —
all while preserving the local-first, no-network-by-default security posture.

## Highlights

- **Policy engine** (`hagrid audit`): evaluate secrets against configurable
  rules for sprawl limits, git tracking, staleness, and vault requirements
- **Watch mode** (`hagrid watch`): persistent file-change monitoring with
  debounced re-scan using OS-level filesystem events
- **Secret rotation** (`hagrid rotate`): format-aware replacement across JSON,
  TOML, .env, and shell files with per-file transaction semantics and rollback
- **Webhook notifications**: explicit-opt-in delivery for drift, policy
  violations, and rotation failures — failure-isolated by construction

## Included Commands

### New in v0.2.0

- `hagrid audit [--json]`
- `hagrid watch`
- `hagrid rotate-info <group-label> [--json]`
- `hagrid rotate <group-label> [--backup]`

### Carried from v0.1.0

- `hagrid init`
- `hagrid scan [--depth lite|standard] [--path <path>] [--json]`
- `hagrid status [--json]`
- `hagrid list [--ungrouped] [--json]`
- `hagrid show <target> [--json]`
- `hagrid suggest [--review] [--json]`
- `hagrid group <label> <ref-id>...`
- `hagrid ungroup <ref-id>`
- `hagrid drift [--json]`
- `hagrid forget <target>`
- `hagrid export [--format json|csv]`

| Code | Meaning |
|------|---------|
| 0 | Success |
| 1 | Fatal runtime error |
| 2 | Usage error |
| 3 | Drift detected (`hagrid drift`) |
| 4 | Policy violation (`hagrid audit`) |
| 5 | Partial rotation failure (`hagrid rotate`) |

## Security Notes

- No plaintext secret values stored in index — only keyed HMAC fingerprints
- No network calls by default — webhooks require explicit opt-in via
  `~/.hagrid/notifications.toml` with `enabled = true`
- Notification payloads exclude secret values, identity keys, and member
  fingerprints
- Notification delivery failures log to stderr and never alter command exit
  codes
- Database encrypted at rest with SQLCipher
- macOS Keychain used for master secret storage
- Rotation uses secure value input via `rpassword` with `Zeroizing<String>`

## Known Limitations

- macOS-only (Linux Keyring support tracked for v0.4+)
- `require_vault` policy rule returns a warning stub (implementation deferred)
- Watch mode does not emit notifications (intentionally deferred to avoid
  coupling a daemon loop to network behavior)
- No retry loop for webhook delivery — best-effort only
- Rotation requires interactive terminal input (no batch/headless mode)
- TUI and MCP server planned for v0.3

## Upgrade from v0.1.0

No database migration required. The v0.2.0 release is backwards-compatible
with existing `~/.hagrid/` data directories created by v0.1.0.

New features require additional config files:
- `~/.hagrid/policies.toml` for `hagrid audit`
- `~/.hagrid/notifications.toml` for webhook notifications

Both are optional and default to no-op when absent.
